# 当前已实现项目结构与流程说明

本文档只描述当前仓库里已经实现的内容。

不包含：

- 尚未实现的未来规划
- 仅在讨论中出现、但还没落地的方案
- 对后续算法阶段的设想

## 1. 当前项目已经实现了什么

截至目前，项目已经实现的是一个“环境与展示底座”，具体包括：

- 二维连续海域场景
- 从左到右的三区带海域
- 伪随机障碍环境生成
- 中间风险区障碍与两条不规则可通过航道约束
- 远海区孤立障碍与风险区
- 近海区基础监测点生成
- 远海区任务热点生成
- HTML 海图输出
- 海图上的静态 `3 个 USV + 2 个 UAV` 外观示意

截至目前，项目还没有实现的是：

- 智能体真实运动
- 路径规划
- 覆盖计算
- 连续空间到离散栅格的覆盖映射
- 协同决策与任务分配

## 2. 当前项目结构是什么意思

### 2.1 海域场景

当前系统里已经有一个固定海域场景：

- 整体大小：`1000m x 1000m`
- 近海区：`0m - 250m`
- 中间风险区：`250m - 450m`
- 远海区：`450m - 1000m`

这部分的作用是：

- 给整个项目提供统一的仿真空间
- 给障碍、智能体外观和后续任务提供共同底图

### 2.2 障碍环境

当前系统已经能生成一套障碍环境。

当前障碍结构包括：

- 中间风险区 `4` 个不规则障碍块
- 中间风险区至少 `2` 条不规则可通过航道约束
- 远海区 `3` 个孤立障碍
- 远海区 `2` 个风险区

这部分的作用是：

- 让海域不再是纯净空白地图
- 先把真实海洋环境中的复杂因素抽象出来
- 给后续路径规划和覆盖任务准备环境基础

当前障碍环境支持伪随机生成，也支持用固定 `seed` 复现同一张环境。

补充说明：

- 当前这版障碍环境已经落地到代码
- 但后续在接入 `USV` 路径规划与动态仿真联调时，这一部分仍可能继续调整

### 2.3 监测点与任务热点

当前系统已经能生成两类静态任务点位：

- 近海区 `2` 个基础监测点
- 远海区 `5` 个任务热点

这些点位当前的生成规则是：

- 都采用伪随机生成
- 会落在对应分区内部
- 会主动避开远海区障碍和风险区

这部分的作用是：

- 先把后续任务层需要的基础点位放进环境
- 为 `UAV` 先发现热点、`USV` 后续复查的协同流程准备静态目标

### 2.4 智能体外观

当前系统里已经把 `3 个 USV + 2 个 UAV` 以静态示意图的形式放进海图。

这里要注意：

- 现在只是“显示出来”
- 不是“已经开始运动”
- 不是“已经开始执行任务”

这部分的作用是：

- 先确认智能体在海图上的外观、大小和可读性
- 给后续动态仿真提前准备显示层

### 2.5 海图输出

当前系统已经能把海域、障碍和静态智能体一起输出成 HTML 海图。

这部分的作用是：

- 把当前环境层直接展示出来
- 让后续所有动态效果都能在这个底图上继续叠加

## 3. 当前系统实际运行流程

如果只看当前已经实现的系统，流程很简单：

1. 运行项目入口
2. 构建默认海域
3. 构建当前障碍环境
4. 构建基础监测点与任务热点
5. 构建静态智能体外观示意
6. 把海域、障碍、监测点、热点、智能体一起渲染成 HTML
7. 将 HTML 写入输出文件
8. 根据参数决定是否自动打开浏览器

## 4. 当前系统输出的是什么

当前系统运行后，输出的是一张 HTML 海图。

这张海图现在包含：

- 三区带海域
- 中间风险区障碍
- 远海区孤立障碍和风险区
- 近海区基础监测点
- 远海区任务热点
- 静态 `3 个 USV + 2 个 UAV`

所以当前系统本质上还是：

- 环境层
- 障碍层
- 静态展示层

还不是完整的区域覆盖仿真系统。
