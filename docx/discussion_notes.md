用途：

- 记录已经确认的策略

## 1. 项目目标

- 项目名称：面向海洋环境监测的异构多智能体区域覆盖方法研究
- 目标系统：`3 个 USV + 2 个 UAV` 空海异构多智能体协同区域覆盖仿真系统

## 2. 当前已确认策略

本节按后续实现顺序排列。后续编码默认按这里从上往下推进。

### 2.1 海域基础场景

已确认：

- 海域采用二维连续空间建模
- 海域大小为 `1000 m × 1000 m`
- 海域采用从左到右的三区带结构
- 近海区：`0 m - 250 m`
- 中间风险区：`250 m - 450 m`
- 远海区：`450 m - 1000 m`

当前代码结果：

### 2.2 障碍环境建模

状态：`已完成`

已确认：

- 障碍信息是对真实海域物理状况的抽象
- 中间风险区必须保证至少存在 `2` 条不规则可通过航道
- 中间风险区不预定义固定航道
- 障碍数量不是核心目标，核心目标是形成合理的通行约束与路径规划难度
- 远海区设置少量孤立障碍或风险点
- 远海区“孤立障碍”表示局部实体通行障碍，主要回答“能不能过”
- 远海区“风险点”表示局部非实体高风险海域，主要回答“值不值得过、是否应尽量避开”
- 近海区设置 `2` 个低优先级基础监测点
- 远海区设置 `5` 个任务热点
- 基础监测点与任务热点在对应分区内随机生成，并避开障碍物与风险区
- 每次仿真环境采用伪随机生成
- 对 `USV`：障碍作为不可穿越区或高代价区域
- 对 `UAV`：障碍保留区域语义，不直接限制飞越

当前代码结果：

- 已完成中间风险区障碍生成
- 已完成中间风险区两条不规则可通过航道约束实现
- 已完成远海区孤立障碍与风险区生成
- 已完成伪随机障碍环境生成逻辑
- 已完成近海区基础监测点随机生成
- 已完成远海区任务热点随机生成
- 已支持通过 `seed` 复现实验环境

当前实现参数：

- 当前采用两条折线控制的不规则可通过航道
- 上航道当前参考宽度约为 `60 m - 68 m`
- 下航道当前参考宽度约为 `64 m - 72 m`
- 远海区当前生成 `3` 个孤立障碍与 `2` 个风险区
- 近海区当前生成 `2` 个基础监测点
- 远海区当前生成 `5` 个任务热点

后续联调关注：

- 需要在后续 `USV` 动态路径规划接入后继续验证当前航道宽度是否合适
- 需要在后续动态仿真中继续观察障碍边缘形态对局部避障表现的支撑效果
- 后续调试过程中，这一部分仍可能继续修改

### 2.3 离散栅格网络与覆盖映射

状态：`已完成`

已确认：

- 覆盖结果需要映射到离散网络
- 第一版离散网络采用规则矩形栅格
- 栅格尺寸不直接等于智能体本体大小
- 栅格尺寸主要由传感器覆盖分辨率、任务精度和计算复杂度决定
- 智能体本体尺寸与栅格尺寸分离建模
- 第一版栅格边长采用 `25 m`
- 第一版离散栅格网络规模为 `40 x 40`
- 第一版栅格显式保存区域属性、障碍属性、风险区属性、基础监测点属性与任务热点属性
- 网格覆盖判定采用“中心点落入 footprint 即覆盖”
- 不计算 footprint 与整个网格的精确相交面积
- 第一版覆盖映射在现有栅格网络上扩展，不单独建立第二套离散结构

当前代码结果：

- 已完成 `25 m` 规则矩形栅格网络构建
- 已完成连续坐标到栅格索引的映射
- 已完成三区带、障碍、风险区、基础监测点与任务热点的栅格落点
- 已完成基于“中心点落入 footprint 即覆盖”的栅格覆盖映射
- 已完成覆盖次数、覆盖来源和最近覆盖步的动态覆盖状态结构
- 已完成覆盖映射结果的静态可视化测试展示

后续联调关注：

- 当前覆盖映射已能支撑基础统计和静态可视化测试，但尚未接入完整动态仿真主循环

### 2.4 Footprint 与异构性

状态：`已完成`

已确认：

- 第一版统一采用圆形 `footprint`
- `USV` 与 `UAV` 通过不同覆盖半径体现异构性
- 默认满足 `UAV 覆盖半径 > USV 覆盖半径`
- 第一版正式覆盖半径设定为：`USV = 50 m`，`UAV = 100 m`
- 第一版覆盖半径与栅格边长的关系为：`USV = 2 个格长`，`UAV = 4 个格长`

当前代码结果：

- 已完成正式圆形 `footprint` 半径参数固化
- 已完成 `USV = 50 m`、`UAV = 100 m` 的代码对齐
- 已完成海图中的圆形 `footprint` 可视化展示

后续联调关注：

- 当前 `footprint` 已使用正式半径，但后续仍需结合动态仿真观察其视觉密度和任务表现

### 2.5 海图与智能体外观

状态：`已完成`

已确认：

- 当前系统已具备 HTML 海图展示能力
- 当前海图已调整为适合后续叠加智能体、轨迹和覆盖结果的底图
- 当前 HTML 中已加入 `3 个 USV + 2 个 UAV` 的静态外观示意
- 海图背景采用白色
- 分区只用浅灰边界和轻标签表示
- 风险区采用浅红轮廓或淡红阴影
- 障碍采用深灰实体
- `USV` 覆盖采用浅蓝
- `UAV` 覆盖采用浅青绿
- 重叠覆盖采用浅金色
- 海图显示模式拆分为 `clean/debug` 两种固定模式
- 同一 HTML 内需要支持 `clean/debug` 两种模式切换
- `clean` 模式默认用于展示，只保留海域、障碍、风险区、基础监测点、热点与智能体本体
- `debug` 模式用于静态验证，额外显示静态覆盖预览与 `footprint` 轮廓
- “静态覆盖预览”统一定义为 `footprint -> 栅格覆盖` 的验证视图，不代表动态仿真结果
- 当前海图中需要预留独立轨迹层，位置位于 `footprint` 之上、本体与标签之下
- 当前海图需要提供智能体编号标签开关

### 2.7 智能体动态仿真

状态：`待实现`

已确认：

- 智能体在连续二维海域中运动、规划与协同
- 后续任务热点应主要由 `UAV` 先行检测
- `USV` 在自身探测过程中也可以独立发现任务热点
- `UAV` 发现任务热点后，再引导 `USV` 前往复查或精细执行
- 在仿真中通过不同探测半径、不同发现能力和不同确认能力体现 `UAV/USV` 的异构热点发现机制
- `UAV` 更适合将热点从“未发现”推进到“疑似热点”
- `USV` 更适合将热点从“疑似热点”推进到“已确认热点”
- 热点状态后续按 `未发现 -> 疑似热点 -> 已确认热点` 的流程推进
- 近海区不作为热点主区域，但保留 `2` 个低优先级基础监测点
- 远海区作为主要任务热点区，设置 `5` 个任务热点

当前未完成：

- 真实运动更新
- 路径跟踪
- 任务推进
- 覆盖执行
